Efficient Movement with Emacs

<div><p><a href="https://www.youtube.com/watch?v=wT_jUtEGfeg">Watch the video</a> on YouTube! </p> <h2><a id="what-will-we-learn" class="anchor" href="#what-will-we-learn">¶</a>What will we learn?</h2><p>We covered the basics of using Emacs in The Absolute Beginners' Guide to Emacs but we didn't really get a chance to talk about the key bindings you might want to use to navigate around file buffers. </p> <p>In this video I'll show you the built-in bindings you can use to move around in a file very efficiently without moving your fingers from the letter keys! </p> <p>I will also tell you the exact Emacs commands that these keys bind to so that you can bind them to new keys if you like or even use them directly in Emacs Lisp code! </p> <p>I recommend opening a text buffer in Emacs while watching this episode so that you can try these bindings and follow along. </p> <h2><a id="moving-the-point" class="anchor" href="#moving-the-point">¶</a>Moving the Point</h2><p>In Emacs, the term "point" refers to the location of the cursor in a given buffer. The point is the place where text will be inserted when you type and its location can be changed using movement keys. </p> <p>Emacs allows you to use the arrow keys to move the point just like in many typical text editors. </p> <p>In addition to the arrow keys, Emacs also provides a set of movement keys using alphabetical letters. The interesting thing is that these keys are chosen because of the words they map to so that they are easy to remember! </p> <ul><li>The letter <kbd>f</kbd> stands for "forward"</li> <li>The letter <kbd>b</kbd> stands for "backward"</li> <li>The letter <kbd>n</kbd> stands for "next"</li> <li>The letter <kbd>p</kbd> stands for "previous"</li> </ul> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Moving-Point.html">Emacs Manual: Changing the Location of Point</a> </p> <h3><a id="basic-movements" class="anchor" href="#basic-movements">¶</a>Basic Movements</h3><p>Basic point movements are done using the keys I mentioned above in combination with the CTRL key: </p> <ul><li><kbd>C-f</kbd> executes <kbd>forward-char</kbd> to move the point forward by one character</li> <li><kbd>C-b</kbd> executes <kbd>backward-char</kbd> to move the point backward by one character</li> <li><kbd>C-n</kbd> executes <kbd>next-line</kbd> to move the point forward by one line, keeping horizontal position</li> <li><kbd>C-p</kbd> executes <kbd>previous-line</kbd> to move the point backward by one line, keeping horizontal position</li> </ul> <p>The benefit of these bindings is that they are closer to where your fingers already rest on the keyboard. No more moving your right hand to the arrow keys just to navigate around the buffer! </p> <p>The <kbd>&lt;LEFT&gt;</kbd> and <kbd>&lt;RIGHT&gt;</kbd> arrow keys are slightly different in that they call <kbd>left-char</kbd> and <kbd>right-char</kbd> which properly respect the direction right-to-left text! </p> <h2><a id="moving-to-beginning-and-end-of-line" class="anchor" href="#moving-to-beginning-and-end-of-line">¶</a>Moving to Beginning and End of Line</h2><p>It's very common to want to move to the beginning or end of a line of text! </p> <ul><li><kbd>C-a</kbd> executes <kbd>move-beginning-of-line</kbd> which moves the point to the beginning of the line</li> <li><kbd>C-e</kbd> executes <kbd>move-end-of-line</kbd> which moves the point to the end of the line</li> </ul> <p>You can think of this like: </p> <ul><li><kbd>a</kbd> is the <em>beginning</em> of the English alphabet</li> <li><kbd>e</kbd> stands for <em>end</em> of line</li> </ul> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Moving-Point.html">Emacs Manual: Changing the Location of Point</a> </p> <h2><a id="moving-to-beginning-and-end-of-buffer" class="anchor" href="#moving-to-beginning-and-end-of-buffer">¶</a>Moving to Beginning and End of Buffer</h2><p>Sometimes you want to jump to the beginning or end of the buffer quickly! </p> <ul><li><kbd>M-&lt;</kbd> executes <kbd>beginning-of-buffer</kbd> to move the point to the beginning of the buffer</li> <li><kbd>M-&gt;</kbd> executes <kbd>end-of-buffer</kbd> to move the point to the end of the buffer.</li> </ul> <p>If you're using the GUI version of Emacs (not in the terminal) you can also press <kbd>C-&lt;HOME&gt;</kbd> and <kbd>C-&lt;END&gt;</kbd> to accomplish the same thing! </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Moving-Point.html">Emacs Manual: Changing the Location of Point</a> </p> <h2><a id="moving-to-a-specific-line-or-column" class="anchor" href="#moving-to-a-specific-line-or-column">¶</a>Moving to a Specific Line or Column</h2><p>When a compiler tells you an error is on a specific line number, you'll want to know how to get there quickly! </p> <ul><li><kbd>M-g M-g</kbd> (press Alt+g twice!) executes <kbd>goto-line</kbd> which prompts you for the line number</li> <li><kbd>M-g &lt;TAB&gt;</kbd> executes <kbd>move-to-column</kbd> which prompts you for the column number</li> </ul> <p><strong>TIP:</strong> you can easily show line numbers in a buffer by running <kbd>M-x display-line-numbers-mode</kbd>! </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Moving-Point.html">Emacs Manual: Changing the Location of Point</a> </p> <h2><a id="moving-by-words-sentences-and-paragraphs" class="anchor" href="#moving-by-words-sentences-and-paragraphs">¶</a>Moving by Words, Sentences, and Paragraphs</h2><p>There are other bindings that move in larger steps in the buffer, usually corresponding to units that you understand like words, sentences, and paragraphs. </p> <h3><a id="words" class="anchor" href="#words">¶</a>Words</h3><p>To move forward and backward by words using the same keys, use ALT (Meta): </p> <ul><li><kbd>M-f</kbd> executes <kbd>forward-word</kbd> to move the point forward by one word</li> <li><kbd>M-b</kbd> executes <kbd>backward-word</kbd> to move the point backward by one word</li> </ul> <p>Note that there is no equivalent for <kbd>C-n</kbd> and <kbd>C-p</kbd>! </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Words.html">Emacs Manual: Words</a> </p> <h3><a id="paragraphs" class="anchor" href="#paragraphs">¶</a>Paragraphs</h3><p>You can move between paragraphs using the following keys: </p> <ul><li><kbd>M-}</kbd> executes <kbd>forward-paragraph</kbd> to move the point forward by one paragraph</li> <li><kbd>M-{</kbd> executes <kbd>backward-paragraph</kbd> to move the point backward by one paragraph</li> </ul> <p>These keys can also be used in code! The definition of a "paragraph" is basically any section of text that is separated by blank lines (see documentation for the <kbd>paragraph-start</kbd> variable). </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Paragraphs.html">Emacs Manual: Paragraphs</a> </p> <h3><a id="sentences" class="anchor" href="#sentences">¶</a>Sentences</h3><p>You can move between sentences using keys similar to jumping to beginning or end of the line: </p> <ul><li><kbd>M-e</kbd> executes <kbd>forward-sentence</kbd> to move the point forward to the end of the sentence or the next sentence</li> <li><kbd>M-a</kbd> executes <kbd>backward-sentence</kbd> to move the point backward to the beginning of the sentence or the preceding sentence</li> </ul> <p>What qualifies as a sentence is a bit more complex (see documentation for the <kbd>sentence-end</kbd> function). If the normal sentence patterns aren't found, it defaults to following paragraph boundaries, so it works in code too! </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Sentences.html">Emacs Manual: Sentences</a> </p> <h2><a id="scrolling-the-window" class="anchor" href="#scrolling-the-window">¶</a>Scrolling the Window</h2><p>Aside from using the expected <kbd>&lt;PageUp&gt;</kbd> and <kbd>&lt;PageDown&gt;</kbd> keys for scrolling the window, you can also use these keys: </p> <ul><li><kbd>C-v</kbd> executes <kbd>scroll-up-command</kbd> to move the window to the next page of content in the buffer</li> <li><kbd>M-v</kbd> executes <kbd>scroll-down-command</kbd> to move the window to the previous page of content in the buffer</li> </ul> <p>The reversed up/down terminology is a little confusing, but it's a more literal description of the direction the buffer contents are moving. </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Scrolling.html">Emacs Manual: Scrolling</a> </p> <h2><a id="centering-the-window" class="anchor" href="#centering-the-window">¶</a>"Centering" the Window</h2><p>These key bindings can be helpful to move the scroll position to give you context above and below the current line: </p> <ul><li><kbd>C-l</kbd> executes <kbd>recenter-top-bottom</kbd> which scrolls the buffer so that the line where the point is positioned in the center of the window. If you press it repeatedly it will cycle so that the current line will show at the top of the window, then the bottom of the window, then at the center again!</li> <li><kbd>C-M-l</kbd> executes <kbd>reposition-window</kbd> which will attempt to scroll the screen to fit as much of the current "thing" as possible in the window (current paragraph, current function definition, etc)</li> </ul> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Recentering.html">Emacs Manual: Recentering</a> </p> <h2><a id="searching-for-text" class="anchor" href="#searching-for-text">¶</a>Searching for Text</h2><p>You can easily move around in the buffer by searching for text with the following bindings: </p> <ul><li><kbd>C-s</kbd> executes <kbd>isearch-forward</kbd> which prompts for the search term and places the cursor after the occurrence found after the point position after you press <kbd>&lt;RET&gt;</kbd> (Enter)</li> <li><kbd>C-r</kbd> executes <kbd>isearch-backward</kbd> which prompts for the search term and places the cursor before the occurrence found before the point position after you press <kbd>&lt;RET&gt;</kbd></li> </ul> <p>These commands are incremental which means that you see the results as you type! </p> <p>If you keep the prompt open (without pressing <kbd>&lt;RET&gt;</kbd>), you can keep pressing the <kbd>C-s</kbd> and <kbd>C-r</kbd> keys to navigate forward and backward in all the results for this string in the same buffer! </p> <p><strong>Tips</strong>: </p> <ul><li>These commands can be useful to just jump forward or backward to a specific character or string on the same line!</li> <li>You can also set a mark just before searching to make it easy to select a specific region.</li> </ul> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/emacs/Basic-Isearch.html">Emacs Manual: Basics of Incremental Search</a> </p> <h2><a id="whats-next" class="anchor" href="#whats-next">¶</a>What's next?</h2><p>In the next video, we'll talk about how you can select regions of text using Emacs' "mark" commands! </p> <p>After that, we'll continue covering the fundamental key bindings for editing and manipulating text. </p> </div>
