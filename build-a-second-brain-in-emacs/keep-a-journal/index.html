Keep a Journal in Emacs with Org Roam

<div><p><a href="https://www.youtube.com/watch?v=3-sLBaJAtew">Watch the video</a> on YouTube! </p> <h2><a id="intro" class="anchor" href="#intro">¶</a>Intro</h2><p>One of the best ways to think and work through problems is to have a written dialogue with yourself. By the end of this video, you'll know how to log your thoughts from the day, link them to existing notes, and easily review the notes for any day, all with Org Roam. </p> <h2><a id="the-starting-configuration" class="anchor" href="#the-starting-configuration">¶</a>The starting configuration</h2><p>Before we start, make sure you've got Org Roam set up already. </p> <p>You can get started quickly by copying the configuration below, but you'll learn more if you watch the two previous videos in the Org Roam series first: </p> <ul><li><a href="https://youtu.be/AyhPmypHDEw">Getting Started with Org Roam</a></li> <li><a href="https://youtu.be/YxgA5z2R08I">Capturing Notes Efficiently with Org Roam</a></li> </ul> <pre><code class="emacs-lisp"> (<span class="org-keyword">use-package</span> <span class="org-constant">org-roam</span> <span class="org-builtin">:ensure</span> t <span class="org-builtin">:init</span> (<span class="org-keyword">setq</span> org-roam-v2-ack t) <span class="org-builtin">:custom</span> (org-roam-directory <span class="org-string">"~/RoamNotes"</span>) (org-roam-completion-everywhere t) <span class="org-builtin">:bind</span> ((<span class="org-string">"C-c n l"</span> . org-roam-buffer-toggle) (<span class="org-string">"C-c n f"</span> . org-roam-node-find) (<span class="org-string">"C-c n i"</span> . org-roam-node-insert) <span class="org-builtin">:map</span> org-mode-map (<span class="org-string">"C-M-i"</span> . completion-at-point) <span class="org-builtin">:map</span> org-roam-dailies-map (<span class="org-string">"Y"</span> . org-roam-dailies-capture-yesterday) (<span class="org-string">"T"</span> . org-roam-dailies-capture-tomorrow)) <span class="org-builtin">:bind-keymap</span> (<span class="org-string">"C-c n d"</span> . org-roam-dailies-map) <span class="org-builtin">:config</span> (<span class="org-keyword">require</span> '<span class="org-constant">org-roam-dailies</span>) <span class="org-comment-delimiter">;; </span><span class="org-comment">Ensure the keymap is available</span> (org-roam-db-autosync-mode)) </code></pre> <h3><a id="configuration-notes" class="anchor" href="#configuration-notes">¶</a>Configuration Notes</h3><ul><li>Make sure to update the value of <kbd>org-roam-directory</kbd> to the path where you'd like your Org Roam files to be stored. Create the directory before loading the configuration!</li> <li>There is one slight difference with this configuration from the one in the previous episodes, the function <kbd>org-roam-setup</kbd> was renamed to <kbd>org-roam-db-autosync-mode</kbd></li> </ul> <h2><a id="creating-the-folder-for-journal-entries" class="anchor" href="#creating-the-folder-for-journal-entries">¶</a>Creating the folder for journal entries</h2><p>Before we can create and view journal entries, we first need to make sure the folder exists. </p> <p>By default, Org Roam will create daily files under the <kbd>daily/</kbd> subfolder of your <kbd>org-roam-directory</kbd>. Let's create that folder! </p> <p>If you'd like to change this folder to a different path, you can set the <kbd>org-roam-dailies-directory</kbd> like so: </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">setq</span> org-roam-dailies-directory <span class="org-string">"journal/"</span>) </code></pre> <p>This path must be relative to <kbd>org-roam-directory</kbd>! </p> <h2><a id="creating-a-journal-entry-for-today" class="anchor" href="#creating-a-journal-entry-for-today">¶</a>Creating a journal entry for today</h2><p>Once you've got Org Roam configured, you can run the command <kbd>org-roam-dailies-capture-today</kbd> which we've bound to <kbd>C-c n d n</kbd>. </p> <p>We're using the existing <kbd>org-roam-dailies-map</kbd> for key bindings! </p> <p>When you run that command, a capture window will appear for a file named with the current date. Here, you can write whatever you like and then press <kbd>C-c C-c</kbd> to save the entry or <kbd>C-c C-k</kbd> to cancel the capture without saving it. </p> <p>The nice thing about this command is that you can run it again to capture another entry, basically allowing you to create entries across the entire day. Let's try it! </p> <p>One major benefit of using Org Roam for your journal (instead of something like <a href="https://github.com/bastibe/org-journal">org-journal</a>) is that you can link to any other note in your Org Roam database. We can use <kbd>C-c n i</kbd> to insert a link to the <kbd>Scheme</kbd> entry that we created in a previous episode. </p> <h2><a id="reviewing-todays-journal-entries" class="anchor" href="#reviewing-todays-journal-entries">¶</a>Reviewing today's journal entries</h2><p>Perhaps you want to take a look at the journal entries you've already written for today. To do this, you can run the command <kbd>org-roam-dailies-goto-today</kbd> which we've bound to <kbd>C-c n d d</kbd>. </p> <p>When you run this command, the journal file for today will be opened so that you can read it and make any edits you like. </p> <p>This file is set up like any other Org Roam note. You can also use <kbd>C-c n f</kbd> to navigate to it and even insert a link to it in other note files! </p> <h2><a id="creating-and-viewing-entries-for-yesterday-or-tomorrow" class="anchor" href="#creating-and-viewing-entries-for-yesterday-or-tomorrow">¶</a>Creating and viewing entries for yesterday or tomorrow</h2><p>You can also create entries for the previous or next day from today: </p> <ul><li>Capture a note for yesterday with <kbd>org-roam-dailies-capture-yesterday</kbd> (<kbd>C-c n d Y</kbd>)</li> <li>Capture a note for tomorrow with <kbd>org-roam-dailies-capture-tomorrow</kbd> (<kbd>C-c n d T</kbd>)</li> </ul> <p>These commands can be useful for writing concluding thoughts for the previous day or leaving notes for yourself for tomorrow! </p> <p>There are also commands to navigate to the entries for these days: </p> <ul><li>Go to the file for yesterday with <kbd>org-roam-dailies-goto-yesterday</kbd> (<kbd>C-c n d y</kbd>)</li> <li>Go to the file for tomorrow with <kbd>org-roam-dailies-goto-tomorrow</kbd> (<kbd>C-c n d t</kbd>)</li> </ul> <h2><a id="creating-and-viewing-entries-for-a-specific-date" class="anchor" href="#creating-and-viewing-entries-for-a-specific-date">¶</a>Creating and viewing entries for a specific date</h2><p>You can create or view notes for a specific day with the following commands: </p> <ul><li>Capture a note for a specific date with <kbd>org-roam-dailies-capture-date</kbd> (<kbd>C-c n d v</kbd>)</li> <li>Go to the file for a specific date with <kbd>org-roam-dailies-goto-date</kbd> (<kbd>C-c n d c</kbd>)</li> </ul> <p>When you run either of these commands, you'll be shown a calendar which lets you pick the date to view or create entries. Just hold shift and use the arrow keys to move around in the calendar! </p> <p>You can also navigate forward or backward in time from the date of the file you're currently viewing: </p> <ul><li>Go back one existing date file with <kbd>org-roam-dailies-goto-next-note</kbd> (<kbd>C-c n d b</kbd>)</li> <li>Go forward one existing date file with <kbd>org-roam-dailies-goto-previous-note</kbd> (<kbd>C-c n d f</kbd>)</li> </ul> <h2><a id="adding-the-time-to-each-entry" class="anchor" href="#adding-the-time-to-each-entry">¶</a>Adding the time to each entry</h2><p>If you want to add some more information to the entries that you capture, you can customize the default capture template: </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">setq</span> org-roam-dailies-capture-templates '((<span class="org-string">"d"</span> <span class="org-string">"default"</span> entry <span class="org-string">"* %&lt;%I:%M %p&gt;: %?"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y-%m-%d&gt;.org"</span> <span class="org-string">"#+title: %&lt;%Y-%m-%d&gt;\n"</span>)))) </code></pre> <p>More information on the format can be found in the documentation for <kbd>format-time-string</kbd>. </p> <p>Now when we create a new daily entry, the date will be captured in the heading. </p> <p>You can create more capture templates for daily entries! I'll give some more examples on how to use this in another video. </p> <h2><a id="the-final-configuration" class="anchor" href="#the-final-configuration">¶</a>The final configuration</h2><p>Here's the final configuration for everything you saw in the video: </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">use-package</span> <span class="org-constant">org-roam</span> <span class="org-builtin">:ensure</span> t <span class="org-builtin">:init</span> (<span class="org-keyword">setq</span> org-roam-v2-ack t) <span class="org-builtin">:custom</span> (org-roam-directory <span class="org-string">"~/RoamNotes"</span>) (org-roam-completion-everywhere t) (org-roam-dailies-capture-templates '((<span class="org-string">"d"</span> <span class="org-string">"default"</span> entry <span class="org-string">"* %&lt;%I:%M %p&gt;: %?"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y-%m-%d&gt;.org"</span> <span class="org-string">"#+title: %&lt;%Y-%m-%d&gt;\n"</span>)))) <span class="org-builtin">:bind</span> ((<span class="org-string">"C-c n l"</span> . org-roam-buffer-toggle) (<span class="org-string">"C-c n f"</span> . org-roam-node-find) (<span class="org-string">"C-c n i"</span> . org-roam-node-insert) <span class="org-builtin">:map</span> org-mode-map (<span class="org-string">"C-M-i"</span> . completion-at-point) <span class="org-builtin">:map</span> org-roam-dailies-map (<span class="org-string">"Y"</span> . org-roam-dailies-capture-yesterday) (<span class="org-string">"T"</span> . org-roam-dailies-capture-tomorrow)) <span class="org-builtin">:bind-keymap</span> (<span class="org-string">"C-c n d"</span> . org-roam-dailies-map) <span class="org-builtin">:config</span> (<span class="org-keyword">require</span> '<span class="org-constant">org-roam-dailies</span>) <span class="org-comment-delimiter">;; </span><span class="org-comment">Ensure the keymap is available</span> (org-roam-db-autosync-mode)) </code></pre> </div>
