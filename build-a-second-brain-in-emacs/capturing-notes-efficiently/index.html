Capturing Notes Efficiently with Org Roam

<div><p><a href="https://youtu.be/YxgA5z2R08I">Watch the video</a> on YouTube! </p> <h2><a id="intro" class="anchor" href="#intro">¶</a>Intro</h2><p>When building a personal knowledge system, one of the most important things you can do is create an efficient workflow for capturing new information. </p> <p>By the end of video you'll know how to create custom templates for capturing notes and project details into your Org Roam files. </p> <h2><a id="the-need-for-a-good-capture-workflow" class="anchor" href="#the-need-for-a-good-capture-workflow">¶</a>The need for a good capture workflow</h2><p>In the previous video, I gave an overview of using a package called Org Roam to build a personal knowledgebase in Org Mode. </p> <p>As you start to use Org Roam to store notes and information, it's useful to figure out the common details that you usually fill in for the things you capture and then create templates to streamline the process. </p> <p>For example, if you often capture detail about books or research papers, there's probably a few different types of information that you'll want to include when you create a new note: </p> <ul><li>Summary of the content</li> <li>Bibliographic information</li> <li>Quotes you might want to use</li> <li>Links to topic notes where you expand on ideas you gained</li> </ul> <p>Org Roam provides a way to define templates for the information you capture so that you'll never forget the useful things you need to include in a new note. </p> <h2><a id="how-org-roam-note-templates-work" class="anchor" href="#how-org-roam-note-templates-work">¶</a>How Org Roam note templates work</h2><p>Org Roam includes its own specialization of Org Mode capture templates. If you've never used capture templates before, don't worry, creating and using Org Roam templates is pretty straightforward. </p> <p>The thing to understand about Org Roam's templates is that they're used when you create a new note. In the last video I showed you how to create a new note with the <kbd>org-roam-node-find</kbd> command which we bound to <kbd>C-c n f</kbd>. </p> <p>By default when you create a new note with this command, Org Roam will automatically create a <kbd>.org</kbd> file with a title and some metadata but nothing else in it. </p> <p>But after you define your own custom note templates, you will be prompted for which template to use when you create a new note. This is how you can decide what type of note will be created and the initial content that it contains! </p> <p><a href="https://www.orgroam.com/manual.html#The-Templating-System">Org Roam Manual: The Templating System</a> </p> <h2><a id="understanding-the-default-template" class="anchor" href="#understanding-the-default-template">¶</a>Understanding the default template</h2><p>To understand Org Roam templates, let's start by looking at the default capture template. We'll do this by setting the <kbd>org-roam-capture-templates</kbd> variable in the Org Roam configuration that we built in the last video. </p> <p>This variable stores the list of capture templates that are used when a new note is created. If you set this variable, you'll have to copy over the default template, otherwise it won't be available anymore. Read its documentation for more details about the format! </p> <p>Here's what the default template looks like: </p> <pre><code class="emacs-lisp"> '((<span class="org-string">"d"</span> <span class="org-string">"default"</span> plain <span class="org-string">"%?"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y%m%d%H%M%S&gt;-${slug}.org"</span> <span class="org-string">"#+title: ${title}\n"</span>) <span class="org-builtin">:unnarrowed</span> t)) </code></pre> <p>Here's what it contains: </p> <ul><li><kbd>"d"</kbd>: The "key", a letter that you press to choose the template</li> <li><kbd>"default"</kbd>: The full name of the template</li> <li><kbd>plain</kbd>: The type of text being inserted, always this value for note templates</li> <li><kbd>"%?"</kbd>: This is the text that will be inserted into the new note, can be anything!</li> <li><kbd>:if-new</kbd> : The list that follows this describes how the note file will be created</li> <li><kbd>:unnarrowed t</kbd> : Ensures that the full file will be displayed when captured (an Org thing)</li> </ul> <p>Aside from the template key and name, the two things you might be interested to change are the initial note text and the format of the filename that gets created, though I don't recommend changing the filename format! </p> <p>The initial note text has its own special format which can be read about in the Org Mode documentation: </p> <p><a href="https://orgmode.org/manual/Template-expansion.html#Template-expansion">Org Mode Manual: Template Expansion</a> </p> <h3><a id="the-complete-starting-configuration" class="anchor" href="#the-complete-starting-configuration">¶</a>The complete starting configuration</h3><pre><code class="emacs-lisp"> (<span class="org-keyword">use-package</span> <span class="org-constant">org-roam</span> <span class="org-builtin">:ensure</span> t <span class="org-builtin">:init</span> (<span class="org-keyword">setq</span> org-roam-v2-ack t) <span class="org-builtin">:custom</span> (org-roam-directory <span class="org-string">"~/RoamNotes"</span>) (org-roam-completion-everywhere t) (org-roam-capture-templates '((<span class="org-string">"d"</span> <span class="org-string">"default"</span> plain <span class="org-string">"%?"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y%m%d%H%M%S&gt;-${slug}.org"</span> <span class="org-string">"#+title: ${title}\n"</span>) <span class="org-builtin">:unnarrowed</span> t))) <span class="org-builtin">:bind</span> ((<span class="org-string">"C-c n l"</span> . org-roam-buffer-toggle) (<span class="org-string">"C-c n f"</span> . org-roam-node-find) (<span class="org-string">"C-c n i"</span> . org-roam-node-insert) <span class="org-builtin">:map</span> org-mode-map (<span class="org-string">"C-M-i"</span> . completion-at-point)) <span class="org-builtin">:config</span> (org-roam-setup)) </code></pre> <h2><a id="creating-a-topic-specific-template" class="anchor" href="#creating-a-topic-specific-template">¶</a>Creating a topic-specific template</h2><p>Let's take a look at a simple example of how to create a new template. We're going to continue with the notes we were using in the previous Org Roam video and add a template for capturing details about a programming language. </p> <pre><code class="emacs-lisp"> (<span class="org-string">"l"</span> <span class="org-string">"programming language"</span> plain <span class="org-string">"* Characteristics\n\n- Family: %?\n- Inspired by: \n\n* Reference:\n\n"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y%m%d%H%M%S&gt;-${slug}.org"</span> <span class="org-string">"#+title: ${title}\n"</span>) <span class="org-builtin">:unnarrowed</span> t) </code></pre> <p>Now let's try creating a new node for "JavaScript" and select our new template. </p> <p>The capture window that appears is filled with the format that we entered into the string! We can type in all the details and save the note with <kbd>C-c C-c</kbd>. </p> <h2><a id="creating-a-literature-reference-template" class="anchor" href="#creating-a-literature-reference-template">¶</a>Creating a literature reference template</h2><p>We can add even more structure to our data entry by using Org Mode's template expansion syntax. </p> <p>The first example of a template we talked about in this video is one to capture details about literature material you read like a book. </p> <p>Let's create a template to capture some of the things we mentioned before. This time, we're going to add custom prompts for certain details we want to fill in like the author and the year the book was published. </p> <pre><code class="emacs-lisp"> (<span class="org-string">"b"</span> <span class="org-string">"book notes"</span> plain <span class="org-string">"\n* Source\n\nAuthor: %^{Author}\nTitle: ${title}\nYear: %^{Year}\n\n* Summary\n\n%?"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y%m%d%H%M%S&gt;-${slug}.org"</span> <span class="org-string">"#+title: ${title}\n"</span>) <span class="org-builtin">:unnarrowed</span> t) </code></pre> <p>Let's create a new note for "Dune" using this template. We'll be prompted for Author and Year which will be inserted in the capture buffer along with the title of the note that we typed in already! The cursor will then jump into the Summary section so that we can type our notes about the book. </p> <p>There are other interesting things you can do with template expansion prompts, so be sure to visit the Org Mode manual to see what else you might be able to use. </p> <p><a href="https://orgmode.org/manual/Template-expansion.html#Template-expansion">Org Mode Manual: Template Expansion</a> </p> <h2><a id="creating-a-project-template" class="anchor" href="#creating-a-project-template">¶</a>Creating a project template</h2><p>Another useful template you might want to create is for capturing details about a new project that you've started, particularly the tasks, goals, and any important dates you might need to remember. </p> <p>Here's a template which sets up a new project file: </p> <pre><code class="emacs-lisp"> (<span class="org-string">"p"</span> <span class="org-string">"project"</span> plain <span class="org-string">"* Goals\n\n%?\n\n* Tasks\n\n** TODO Add initial tasks\n\n* Dates\n\n"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y%m%d%H%M%S&gt;-${slug}.org"</span> <span class="org-string">"#+title: ${title}\n#+filetags: Project"</span>) <span class="org-builtin">:unnarrowed</span> t) </code></pre> <p>The new thing we've added this time is the <kbd>#+filetags: Project</kbd> configuration inside of the <kbd>file+head</kbd> section. This part defines what information gets inserted in the header lines of the new file. </p> <p>What we've done here is add a <kbd>Project</kbd> tag to the file. Tagging your note files gives you some extra benefits when you start using the Org Roam DB API to query your notes. The tag may also show up in your completion list if you're using Vertico and Marginalia, <a href="https://www.youtube.com/watch?v=J0OaRy85MOo">check out my video on Vertico</a> if you haven't seen it yet. </p> <p>An example I'll show you in a later video is how you can look up all your project notes using this tag and automatically build your Org Agenda list from that set of files! </p> <h2><a id="storing-templates-in-org-files" class="anchor" href="#storing-templates-in-org-files">¶</a>Storing templates in Org files</h2><p>Let's talk about how you can write your capture templates in real Org files so that you don't have to edit strings inside of your Emacs configuration. </p> <p>It's actually pretty easy to do this! Just create a new <kbd>.org</kbd> file containing the contents of the template string we created before and then clean it up so that it looks like a normal file. </p> <p>Once you've saved the template file, you'll have to replace the string in the capture template with a <kbd>file</kbd> sub-list which includes the template file path. </p> <p>Here's how we would need to change the book notes template to refer to a template file instead: </p> <pre><code class="emacs-lisp"> (<span class="org-string">"b"</span> <span class="org-string">"book notes"</span> plain (file <span class="org-string">"~/RoamNotes/Templates/BookNoteTemplate.org"</span>) <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y%m%d%H%M%S&gt;-${slug}.org"</span> <span class="org-string">"#+title: ${title}\n"</span>) <span class="org-builtin">:unnarrowed</span> t) </code></pre> <h2><a id="customizing-the-default-template" class="anchor" href="#customizing-the-default-template">¶</a>Customizing the default template</h2><p>You can also customize the default template! </p> <p>Let's try to insert the date when the note was created. We can do this by adding the string <kbd>#+date: %U</kbd> to the initial header string in the <kbd>file+head</kbd> configuration: </p> <pre><code class="emacs-lisp"> (<span class="org-string">"d"</span> <span class="org-string">"default"</span> plain <span class="org-string">"%?"</span> <span class="org-builtin">:if-new</span> (file+head <span class="org-string">"%&lt;%Y%m%d%H%M%S&gt;-${slug}.org"</span> <span class="org-string">"#+title: ${title}\n#+date: %U\n"</span>) <span class="org-builtin">:unnarrowed</span> t) </code></pre> <p>And like I mentioned earlier, you can change the naming of files Org Roam creates if you really want to, just change the first string in the <kbd>file+head</kbd> to contain the format you want where <kbd>${slug}</kbd> is the part of the filename that reflects the initial note title. </p> <p>Keep in mind that you'll need to replicate your <kbd>:if-new</kbd> settings across all your templates if you want consistent file naming and header contents! </p> <h2><a id="whats-next" class="anchor" href="#whats-next">¶</a>What's next?</h2><p>Org Roam provides another useful feature for capturing thoughts and ideas that occur during the day called "dailies". In the next video, I'll show you how you can use this feature for keeping a journal, a daily log of your work, and for any random things you want to remember! </p> </div>
