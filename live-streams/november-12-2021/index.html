<div><p><a href="https://youtu.be/dB4ydMQgmNk">Watch the recording</a> on YouTube! </p> <h2><a id="editor-popularity-does-it-matter" class="anchor" href="#editor-popularity-does-it-matter">¶</a>Editor Popularity: Does it matter?</h2><p>Articles and forum threads with concern about Emacs' perceived lack of popularity keep appearing every year or two. </p> <ul><li><a href="https://lwn.net/Articles/819452/">Making Emacs Popular Again</a></li> <li><a href="https://news.ycombinator.com/item?id=29105032">Recent Hacker News discussion</a></li> </ul> <p>Today we're going to discuss the topic of popularity as it relates to text editors: </p> <ul><li>Does popularity matter?</li> <li>What happens when an editor starts to lose popularity</li> </ul> <p>The goal of this conversation is for us to try answer the following questions: </p> <h3><a id="which-text-editors-have-been-popular-in-the-last-10-15-years" class="anchor" href="#which-text-editors-have-been-popular-in-the-last-10-15-years">¶</a>Which text editors have been popular in the last 10-15 years?</h3><ul><li><a href="https://trends.google.com/trends/explore?date=all&geo=US&q=%2Fm%2F01yp0m,%2Fm%2F07zh7,%2Fm%2F09gl10,%2Fm%2F0b6h18n,%2Fm%2F0134xwrk">Compare Emacs, Vim, TextMate, Sublime Text, VS Code</a></li> <li><a href="https://insights.stackoverflow.com/survey/2021#integrated-development-environment">StackOverflow 2021 IDE Results</a></li> <li><a href="https://blog.robenkleene.com/2020/09/21/the-era-of-visual-studio-code/">https://blog.robenkleene.com/2020/09/21/the-era-of-visual-studio-code/</a></li> <li class="off"> GNU Emacs</li> <li class="on"> Sublime Text</li> <li class="on"> Atom</li> <li class="on"> TextMate</li> <li class="on"> Brackets</li> <li class="off"> Notepad++</li> <li class="off"> XEmacs</li> <li class="off"> BBEdit</li> <li class="off"> Vim / Neovim</li> <li>VS Code</li> <li>Netbeans</li> <li>LightTable</li> <li>IntelliJ / JetBrains Editor</li> <li>Kakoune</li> <li>code:blocks</li> </ul> <h3><a id="what-are-some-examples-of-editors-losing-popularity-and-why-did-it-happen" class="anchor" href="#what-are-some-examples-of-editors-losing-popularity-and-why-did-it-happen">¶</a>What are some examples of editors losing popularity and why did it happen?</h3><h4><a id="textmate" class="anchor" href="#textmate">¶</a>TextMate</h4><p>Other more extensible and featureful editors came along despite TextMate being very influential. </p> <p>Also no UTF-8 support </p> <p>DHH stopped using it? </p> <h4><a id="brackets" class="anchor" href="#brackets">¶</a>Brackets</h4><p>Potentially unseated by Atom because it had more broad scope and extensibility. </p> <h4><a id="sublime-text" class="anchor" href="#sublime-text">¶</a>Sublime Text</h4><p>There were no new updates to Sublime Text 2 and 3 for a long time and then VS Code came long. </p> <p>No built in package manager, extensibility wasn't as good as other editors. </p> <p>Not open source, single developer (or small team at least) and proprietary code. </p> <h4><a id="atom" class="anchor" href="#atom">¶</a>Atom</h4><p>VS Code appeared, lots of hype after it was released, took attention away from Atom </p> <h4><a id="emacs" class="anchor" href="#emacs">¶</a>Emacs</h4><p>More options appearing over time that are more appealing to different types of users </p> <h3><a id="what-happens-when-an-editor-loses-popularity" class="anchor" href="#what-happens-when-an-editor-loses-popularity">¶</a>What happens when an editor loses popularity?</h3><ul><li>Less people contributing to the project</li> <li>No new features being added to stay competitive with newer editors</li> <li>Code rot, bugs and security issue</li> <li>Less people creating and maintaining community packages</li> <li>Less blog posts, videos, etc educating people on how to learn it</li> <li>Not as much discussion about it on forums or chat</li> <li>Might not be packaged anymore in Linux distributions</li> <li>Not as many new users coming in can have a compounding effect on user loss</li> </ul> <h3><a id="how-much-popularity-is-enough-to-keep-an-editor-alive" class="anchor" href="#how-much-popularity-is-enough-to-keep-an-editor-alive">¶</a>How much popularity is enough to keep an editor alive?</h3><p>What are the signs that there is <strong>enough</strong> popularity? </p> <ul><li>Do people still care enough to set up conferences about the editor</li> <li>Plenty of different places where users congregate and chat (Reddit, IRC, Discord, etc)</li> <li>A healthy stream of blog posts or videos that explain how to use it or at least provide perspectives</li> <li>Users that can't stop talking about it, even to people who don't want to hear it</li> </ul> <h3><a id="so-does-popularity-really-matter" class="anchor" href="#so-does-popularity-really-matter">¶</a>So, does popularity really matter?</h3><p>Yes and no, but an editor does not need to be the most popular editor to survive. It only needs dedicated users who care enough to keep moving things forward. </p> <h3><a id="other-points" class="anchor" href="#other-points">¶</a>Other points</h3><ul><li>The type of users can also help keep the editor alive if they can contribute to the project; Emacs users have much more understanding of how to contribute to Emacs as they get better at config hacking and writing packages</li> <li>Emacs hacking scales from simple tweaks to building full packages and even contributing to the core</li> </ul> </div>
