System Crafters Live! - August 27, 2021

<div><p><a href="https://youtu.be/nMH3QXWxTGg">Watch the recording</a> on YouTube! </p> <h2><a id="system-crafters-store-is-live" class="anchor" href="#system-crafters-store-is-live">¶</a>System Crafters Store is live!</h2><p>I've been getting requests for merch for a while, that day has finally arrived! </p> <p>Check out the set of items I've put together with the System Crafters logo: </p> <p><a href="https://store.systemcrafters.net">https://store.systemcrafters.net</a> </p> <p>These are all high quality print-on-demand items so shipping can take up to 2 weeks but it's usually faster than the estimate you get when you buy something. </p> <p>Please let me know your feedback on <em>everything</em>: items, pricing, shipping costs, etc. I want the community to be proud to wear them! </p> <p>This is just the first step; I plan to add new designs every 3 months or so, it will only get better from here! </p> <h2><a id="we-need-to-talk-about-exwm" class="anchor" href="#we-need-to-talk-about-exwm">¶</a>We Need to Talk About EXWM</h2><p>As I've mentioned before a few times, EXWM the maintainer <a href="https://github.com/ch11ng">ch11ng</a> has been missing for over a year. </p> <ul><li>Last commits to EXWM and XELB were in July 2020</li> <li>People are concerned: <a href="https://github.com/ch11ng/exwm/issues/845">https://github.com/ch11ng/exwm/issues/845</a></li> <li>Many lingering issues that people face daily, I've been burned frequently</li> <li>I even tried StumpWM again!</li> <li>Nothing beats EXWM, though.</li> </ul> <p>What can be done about it? </p> <h2><a id="can-the-project-be-revived" class="anchor" href="#can-the-project-be-revived">¶</a>Can the project be revived?</h2><ul><li>The project is in definite need of help to address pain points</li> <li>It seems <a href="https://github.com/medranocalvo">medranocalvo</a> might be an admin on the repo</li> <li>All it would really take is for an individual or small group to learn the code and try fixing things</li> <li>FSF copyright assignment approval is a big challenge (can that be abandoned?)</li> <li>Many people would be happy and relieved for EXWM to be maintained again</li> </ul> <h2><a id="should-someone-fork-exwm" class="anchor" href="#should-someone-fork-exwm">¶</a>Should someone fork EXWM?</h2><ul><li>A hard fork with no intention to merge upstream can avoid FSF assignment issues</li> <li>Could be an opportunity to rethink some things, redesign the code</li> <li>Aside from learning the code, the challenge would be making it enticing (and smooth) to migrate</li> <li>This could be the fastest route to a bright new future</li> </ul> <h2><a id="a-new-alternative" class="anchor" href="#a-new-alternative">¶</a>A new alternative?</h2><p>If we're considering a fork, why not think bigger? </p> <ul><li>Can we fix some core issues with EXWM while planning for the future?</li> <li>Minimal window manager core <em>not</em> written in Emacs Lisp but driven by it via RPC</li> <li>Write it either in C or an embedded Scheme (Chibi, perhaps)</li> <li>Create an abstract internal model; focus on X Window System and Wayland later</li> <li>Make it possible to have a more traditional tiling window manager that is still Emacs-driven</li> <li>OMG GAPS</li> </ul> <p>Could be a huge undertaking, but possibly more future-proof! </p> <h2><a id="heres-what-ill-do" class="anchor" href="#heres-what-ill-do">¶</a>Here's what I'll do</h2><p>Solving this problem is important to me! I can't live without my Emacs Desktop Environment. </p> <ul><li>I'm going to start investigating all possibilities in my (limited) free time</li> <li>Great opportunity to stream the work on Twitch: <a href="https://twitch.tv/SystemCrafters">https://twitch.tv/SystemCrafters</a></li> <li>Start building a knowledgebase of how window management works with Xorg</li> <li>Investigate various technical options, live coding regularly</li> <li>As soon as a good direction is determined, get community involvement</li> </ul> <p>For stream schedule: Mondays or Wednesdays? Tentatively 8am-10am Pacific Time. </p> <h2><a id="what-do-you-think" class="anchor" href="#what-do-you-think">¶</a>What do you think?</h2> <h2><a id="random-chat" class="anchor" href="#random-chat">¶</a>Random Chat</h2><ul><li><a href="https://emacsconf.org/2021/cfp/">EmacsConf CFP</a></li> <li><a href="https://vieb.dev/screenshots">Vieb browser</a></li> <li><a href="https://luakit.github.io/">Luakit browser</a></li> <li><a href="https://fennel-lang.org/">Fennel language</a></li> <li><a href="https://daviwil.com/ld28/">One Sword game</a></li> <li><a href="https://lucidmanager.org/productivity/emacs-windows/">Installing Emacs on Windows 10</a></li> <li><a href="https://github.com/Wilfred/helpful">helpful.el</a></li> <li><a href="https://gitlab.com/ideasman42/emacs-sidecar-locals">emacs-sidecar-locals</a> - or <kbd>dir-locals-set-directory-class</kbd>, etc</li> <li><a href="https://sachachua.com/blog/2021/08/started-learning-how-to-interactively-debug-javascript-in-emacs-with-indium/">Debugging JavaScript in Emacs with Indium</a></li> <li><a href="https://www.youtube.com/watch?v=w3NfOeecMkI">Taking a look at GNU Hurd</a></li> <li><a href="https://takeonrules.com/2021/08/22/ever-further-refinements-of-org-roam-usage/">Even Further Refinements of Org Roam Usage</a></li> <li><a href="https://github.com/a-schaefers/systemE">systemE</a></li> <li><a href="https://protesilaos.com/news/2021-08-16-status-update-computer/">Prot system failure</a> (<a href="https://www.paypal.me/protesilaos">Paypal</a>)</li> </ul> <p>Check out <a href="https://sachachua.com/blog/category/emacs-news/">Emacs News</a>! </p> </div>
