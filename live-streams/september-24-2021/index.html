href="#investigating-use-package-alternatives">¶</a>Investigating use-package Alternatives</h2><p>Today we'll be taking a look at two alternatives to <kbd>use-package</kbd>: <kbd>setup.el</kbd> and <kbd>leaf.el</kbd>. </p> <p>The goal isn't to try and replace <kbd>use-package</kbd>, just to investigate the alternate ways that Emacs users have created to simplify configuration authoring! </p> <h2><a id="comparing-with-kbduse-packagekbd" class="anchor" href="#comparing-with-kbduse-packagekbd">¶</a>Comparing with <kbd>use-package</kbd></h2><p>I'll try to put together configurations with <kbd>setup.el</kbd> and <kbd>leaf.el</kbd> to compare how they would look to an equivalent <kbd>use-package</kbd> configuration. </p> <p>We'll try to see how to replicate the same features that <kbd>use-package</kbd> provides: </p> <ul><li>:ensure - Install package</li> <li>:after - Load after another package</li> <li>:init - Run code before package loads</li> <li>:config - Run code after package loads</li> <li>:custom - Set custom variables with concise syntax</li> <li>:defer - Defer loading until some later point</li> <li>:demand - Ensure loading at startup</li> <li>:bind - Bind keys for modes</li> <li>:hook - Set hooks that will cause package to load</li> <li>:commands - Set autoloaded commands that will cause packag to load</li> <li>:mode - Activate mode when particular file types are opened</li> </ul> <h2><a id="the-example-configuration" class="anchor" href="#the-example-configuration">¶</a>The example configuration</h2><pre><code class="emacs-lisp"> (<span
