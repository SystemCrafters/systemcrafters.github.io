<div><p><a href="https://youtu.be/R5cdtSfTpE0">Watch the recording</a> on YouTube! </p> <h2><a id="news" class="anchor" href="#news">¶</a>News</h2><ul><li>Emacs 28 release branch has been created! Start testing it out now to help find bugs before release: <a href="https://lists.gnu.org/archive/html/emacs-devel/2021-09/msg02290.html">https://lists.gnu.org/archive/html/emacs-devel/2021-09/msg02290.html</a></li> </ul> <h2><a id="what-is-kbdguix-homekbd" class="anchor" href="#what-is-kbdguix-homekbd">¶</a>What is <kbd>guix home</kbd>?</h2><p><kbd>guix home</kbd> allows you to apply configurations to your user account with all the same benefits that you get from <kbd>guix system</kbd>: </p> <ul><li>Install packages</li> <li>Place files in the filesystem (dotfiles?)</li> <li>Configure your programs with Scheme (!)</li> <li>Run user-level services</li> <li>Roll back to previous configurations (!)</li> </ul> <p>One really nice benefit is that it can be used on "foreign" distributions, try it out without fully switching to Guix System! </p> <p>This could easily replace some of the configuration work I do with Org Babel, so it will be interesting to see how this plays out. </p> <p>Check out the <a href="https://guix.gnu.org/manual/devel/en/html_node/Home-Configuration.html">Home Configuration</a> page in the Guix manual for more information. </p> <h3><a id="useful-links" class="anchor" href="#useful-links">¶</a>Useful Links</h3><ul><li>Andrew Tropin's YouTube channel: <a href="https://www.youtube.com/c/TROP1N/">https://www.youtube.com/c/TROP1N/</a></li> <li>Andrew's rde channel/repo: <a href="https://github.com/abcdw/rde">https://github.com/abcdw/rde</a></li> <li>Andrew's rde mailing lists: <a href="https://lists.sr.ht/~abcdw">https://lists.sr.ht/~abcdw</a></li> <li>rde home config example: <a href="https://github.com/abcdw/rde/blob/master/rde/examples/abcdw/configs.scm">https://github.com/abcdw/rde/blob/master/rde/examples/abcdw/configs.scm</a></li> <li>yoctocel's home config: <a href="https://hg.sr.ht/~yoctocell/guixrc/browse/yoctocell/home">https://hg.sr.ht/~yoctocell/guixrc/browse/yoctocell/home</a></li> <li>krevedkokun's home config: <a href="https://git.sr.ht/~krevedkokun/dotfiles/tree/master/item/home/yggdrasil">https://git.sr.ht/~krevedkokun/dotfiles/tree/master/item/home/yggdrasil</a></li> </ul> <h2><a id="lets-try-it" class="anchor" href="#lets-try-it">¶</a>Let's try it!</h2><p>I have not tried to use <kbd>guix home</kbd> yet, so we're going to try it out for the first time together today. </p> <p>I'll try to see how far we can get with building a home configuration in a test environment, including cloning my dotfiles repo and applying some of the files from it. </p> <p>Some questions: </p> <ul><li>What happens when <kbd>.profile</kbd> already exists?</li> <li>Can manifest files be kept separately and loaded into the home config?</li> <li>Can we put system and home configurations into the same file? (I think yes)</li> <li>Is there a way to pin user-level channel configuration easily?</li> <li>How nicely will this play with Org Babel configs or anything else that will be hand-edited outside of the scope of <kbd>guix home</kbd> (like an Emacs config)?</li> </ul> </div>
