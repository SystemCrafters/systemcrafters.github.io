Creating a Custom Minor Mode

<div><p><a href="https://youtu.be/JJqchO3fnds">Watch the video</a> on YouTube! </p> <h2><a id="what-will-we-learn" class="anchor" href="#what-will-we-learn">¶</a>What will we learn?</h2><p>Today we're going to learn about Emacs' concept of modes and then walk through some practical examples of creating a custom minor mode. </p> <p>We'll be continuing with the <kbd>dotcrafter.el</kbd> project that we started working on earlier in this series! At this point we have full functionality for managing our dotfiles configuration both from within Org Mode files and with individual files that must be symbolically linked back into your home directory. </p> <p>The minor mode that we'll write today will enable us to automatically "tangle" our Org-based configuration files each time we edit one of them and then link any new output configuration files into our home directory. We'll also be able to set up a custom key map for dotcrafter commands which will only be activated when the mode is turned on! </p> <h2><a id="what-is-a-mode" class="anchor" href="#what-is-a-mode">¶</a>What is a "mode"?</h2><p>Much of the user-facing functionality in Emacs is provided by <strong>major</strong> and <strong>minor</strong> modes. </p> <ul><li>A <strong>major</strong> mode is the main mode for a buffer, there can only be one active at any time (example: <kbd>org-mode</kbd>)</li> <li>A <strong>minor</strong> mode provides supporting functionality either in a specific buffer or globally in Emacs (example: <kbd>auto-fill-mode</kbd>)</li> </ul> <p>Both major and minor modes can enable certain types of functionality when they are activated: </p> <ul><li>A key map that is only active when the mode is enabled</li> <li>A mode line indicator showing the mode is active</li> <li>A hook that is executed so that other functions or modes can be activated along with the mode</li> <li>Arbitrary code that gets executed when the mode is activated or deactivated</li> </ul> <p>Any mode that is defined will have a command that enables or disables it which can be invoked interactively or in Emacs Lisp: </p> <pre><code class="emacs-lisp"> M-x org-mode (auto-revert-mode 1) </code></pre> <h2><a id="writing-a-basic-minor-mode" class="anchor" href="#writing-a-basic-minor-mode">¶</a>Writing a basic minor mode</h2><p>A minor mode is basically a bit of Emacs Lisp code that conforms to a set of conventions on how minor modes should be defined. Here are the high level things that a custom minor mode needs to work correctly in Emacs: </p> <ul><li>A variable for the mode that ends in <kbd>-mode</kbd> like <kbd>dotcrafter-mode</kbd>. This variable will be set to <kbd>nil</kbd> when the mode is not active or any other value to indicate that it is active. If the minor mode is meant to be buffer-local, the <kbd>make-variable-buffer-local</kbd> function can be used to make the mode variable buffer-local.</li> <li>A command for enabling and disabling the mode which has the same name as the mode variable (e.g. <kbd>dotcrafter-mode</kbd>). It will accept one optional argument which indicates whether the mode should be enabled or disabled, typically <kbd>t</kbd> or a positive number to activate the mode or a non-positive number (0 or less than 0) to deactivate the mode. <p>When the optional parameter is omitted, assume this means that the mode should be activated when the function is called in Emacs Lisp or that the mode should be toggled when invoked interactively by the user. </p></li> <li>If desired, a keymap for the mode can be enabled by adding an entry to the <kbd>minor-mode-map-alist</kbd> which registers. The entry will be a pair containing the minor mode's variable symbol and the keymap to be used for that mode.</li> <li>If desired, a "lighter" for the minor mode can be displayed in the mode line by adding an entry to the <kbd>minor-mode-alist</kbd>. The entry will be a pair containing the minor mode's variable symbol and the string to display in the mode line.</li> </ul> <p>The command that is used for enabling and disabling the minor mode can then make whatever changes are needed to the buffer-local or global state of Emacs to enable the mode's behavior. </p> <p>Note that the mode command will have to check the mode variable to see if it is already enabled before enabling/disabling the mode functionality! </p> <h2><a id="example-creating-a-simple-minor-mode-from-scratch" class="anchor" href="#example-creating-a-simple-minor-mode-from-scratch">¶</a>Example: Creating a simple minor mode from scratch</h2><p>Let's follow the conventions we talked about and write a simple minor mode from scratch! Our minor mode, <kbd>dotcrafter-basic-mode</kbd>, will simply set a keymap that can be used when this mode is active and also update the mode line with the string "dotcrafter" to indicate its state. </p> <pre><code class="emacs-lisp"> (make-variable-buffer-local (<span class="org-keyword">defvar</span> <span class="org-variable-name">dotcrafter-basic-mode</span> nil <span class="org-doc">"Toggle dotcrafter-basic-mode."</span>)) (<span class="org-keyword">defvar</span> <span class="org-variable-name">dotcrafter-basic-mode-map</span> (make-sparse-keymap) <span class="org-doc">"The keymap for dotcrafter-basic-mode"</span>) <span class="org-comment-delimiter">;; </span><span class="org-comment">Define a key in the keymap</span> (define-key dotcrafter-basic-mode-map (kbd <span class="org-string">"C-c C-. t"</span>) (<span class="org-keyword">lambda</span> () (<span class="org-keyword">interactive</span>) (message <span class="org-string">"dotcrafter key binding used!"</span>))) (add-to-list 'minor-mode-alist '(dotcrafter-basic-mode <span class="org-string">" dotcrafter"</span>)) (add-to-list 'minor-mode-map-alist (cons 'dotcrafter-basic-mode dotcrafter-basic-mode-map)) (<span class="org-keyword">defun</span> <span class="org-function-name">dotcrafter-basic-mode</span> (<span class="org-type">&amp;optional</span> ARG) (<span class="org-keyword">interactive</span> (list 'toggle)) (<span class="org-keyword">setq</span> dotcrafter-basic-mode (<span class="org-keyword">if</span> (eq ARG 'toggle) (not dotcrafter-basic-mode) (&gt; ARG 0))) <span class="org-comment-delimiter">;; </span><span class="org-comment">Take some action when enabled or disabled</span> (<span class="org-keyword">if</span> dotcrafter-basic-mode (message <span class="org-string">"dotcrafter-basic-mode activated!"</span>) (message <span class="org-string">"dotcrafter-basic-mode deactivated!"</span>))) </code></pre> <p>Note that this doesn't perfectly follow all of the conventions: minor mode commands should also accept numeric prefix arguments for controlling whether the mode gets enabled or disabled, but it's not a hard requirement. </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/elisp/Minor-Mode-Conventions.html">Emacs Lisp Manual: Conventions for Writing Minor Modes</a> </p> <h2><a id="creating-a-hook" class="anchor" href="#creating-a-hook">¶</a>Creating a Hook</h2><p>One useful thing you might want to provide with your minor mode is a hook. A hook is a variable that stores a list of functions to be invoked when <em>something</em> happens, like when a mode is activated. </p> <p>Defining a hook is easy, you just define a variable that is set to <kbd>nil</kbd> or an empty list: </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">defvar</span> <span class="org-variable-name">dotcrafter-basic-mode-hook</span> nil <span class="org-doc">"The hook for dotcrafter-basic-mode."</span>) </code></pre> <p>To execute a hook, use the <kbd>run-hooks</kbd> function: </p> <pre><code class="emacs-lisp"> (run-hooks 'dotcrafter-basic-mode-hook) </code></pre> <p>Right now it doesn't do anything! We need to add a function to the hook with <kbd>add-hook</kbd>: </p> <pre><code class="emacs-lisp"> (add-hook 'dotcrafter-basic-mode-hook (<span class="org-keyword">lambda</span> () (message <span class="org-string">"Hook was executed!"</span>))) </code></pre> <p>Now we can update our basic minor mode to invoke this hook: </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">defun</span> <span class="org-function-name">dotcrafter-basic-mode</span> (<span class="org-type">&amp;optional</span> ARG) (<span class="org-keyword">interactive</span> (list 'toggle)) (<span class="org-keyword">setq</span> dotcrafter-basic-mode (<span class="org-keyword">if</span> (eq ARG 'toggle) (not dotcrafter-basic-mode) (&gt; ARG 0))) <span class="org-comment-delimiter">;; </span><span class="org-comment">Take some action when enabled or disabled</span> (<span class="org-keyword">if</span> dotcrafter-basic-mode (message <span class="org-string">"dotcrafter-basic-mode activated!"</span>) (message <span class="org-string">"dotcrafter-basic-mode deactivated!"</span>)) <span class="org-comment-delimiter">;; </span><span class="org-comment">Run any registered hooks</span> (run-hooks 'dotcrafter-basic-mode-hook)) </code></pre> <p>You can define as many hooks as you need for your minor mode! It doesn't have to just be the hook for when the mode gets activated. </p> <p><strong>NOTE:</strong> Major modes should use the <kbd>run-mode-hooks</kbd> function instead! There is extra logic to be used in that case. <kbd>run-hooks</kbd> is fine for minor mode hooks. </p> <h2><a id="using-the-kbddefine-minor-modekbd-macro" class="anchor" href="#using-the-kbddefine-minor-modekbd-macro">¶</a>Using the <kbd>define-minor-mode</kbd> macro</h2><p>Learning how to create your own minor mode from scratch is useful to show how simple they really are in practice, but once you understand it, you shouldn't have to do it that way every time. </p> <p>It is much easier to use the <kbd>define-minor-mode</kbd> macro to define a minor mode since it does most of the work for you! </p> <p>Here's an example of defining a mode called <kbd>dotcrafter-mode</kbd> that is exactly like <kbd>dotcrafter-basic-mode</kbd> except for how it is defined: </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">define-minor-mode</span> <span class="org-function-name">dotcrafter-mode</span> <span class="org-doc">"Toggles global dotcrafter-mode."</span> nil <span class="org-comment-delimiter">; </span><span class="org-comment">Initial value, nil for disabled</span> <span class="org-builtin">:global</span> t <span class="org-builtin">:group</span> 'dotfiles <span class="org-builtin">:lighter</span> <span class="org-string">" dotcrafter"</span> <span class="org-builtin">:keymap</span> (list (cons (kbd <span class="org-string">"C-c C-. t"</span>) (<span class="org-keyword">lambda</span> () (<span class="org-keyword">interactive</span>) (message <span class="org-string">"dotcrafter key binding used!"</span>)))) (<span class="org-keyword">if</span> dotcrafter-mode (message <span class="org-string">"dotcrafter-basic-mode activated!"</span>) (message <span class="org-string">"dotcrafter-basic-mode deactivated!"</span>))) (add-hook 'dotcrafter-mode-hook (<span class="org-keyword">lambda</span> () (message <span class="org-string">"Hook was executed!"</span>))) (add-hook 'dotcrafter-mode-on-hook (<span class="org-keyword">lambda</span> () (message <span class="org-string">"dotcrafter turned on!"</span>))) (add-hook 'dotcrafter-mode-off-hook (<span class="org-keyword">lambda</span> () (message <span class="org-string">"dotcrafter turned off!"</span>))) </code></pre> <p>What we get: </p> <ul><li><kbd>dotcrafter-mode</kbd> command</li> <li><kbd>dotcrafter-mode</kbd> variable</li> <li><kbd>dotcrafter-mode-hook</kbd> (also <kbd>dotcrafter-mode-on-hook</kbd> and <kbd>dotcrafter-mode-off-hook</kbd>)</li> <li><kbd>dotcrafter-mode-map</kbd></li> <li>Lighter and mode map registration</li> </ul> <p>Let's take a look at the generated <kbd>dotcrafter-mode</kbd> command using <kbd>C-h f</kbd>! </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_node/elisp/Defining-Minor-Modes.html">Emacs Lisp Manual: Defining Minor Modes</a> </p> <h2><a id="example-writing-real-behavior-for-kbddotcrafter-modekbd" class="anchor" href="#example-writing-real-behavior-for-kbddotcrafter-modekbd">¶</a>Example: Writing real behavior for <kbd>dotcrafter-mode</kbd></h2><p>Now let's take what we've learned and set up <kbd>dotcrafter-mode</kbd> so that we will automatically tangle and update configuration target files for any Org Mode file that lives inside of our dotfiles folder. </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">defcustom</span> <span class="org-variable-name">dotcrafter-keymap-prefix</span> <span 'dotfiles) (<span class="org-keyword">defcustom</span> <span class="org-variable-name">dotcrafter-tangle-on-save</span> t <span class="org-doc">"When t, automatically tangle Org files on save."</span> class="org-function-name">dotcrafter-tangle-org-file</span> (<span class="org-type">&amp;optional</span> org-file) <span class="org-doc">"Tangles a single .org file relative to the path in</span> <span class="org-doc">dotfiles-folder. If no file is specified, tangle the current</span> <span class="org-doc">file if it is an org-mode buffer inside of dotfiles-folder."</span> (<span class="org-keyword">interactive</span>) <span class="org-comment-delimiter">;; </span><span class="org-comment">Suppress prompts and messages</span> (<span class="org-keyword">let</span> ((org-confirm-babel-evaluate nil) (message-log-max nil) (inhibit-message t)) (org-babel-tangle-file (expand-file-name org-file dotcrafter-dotfiles-folder)) <span class="org-comment-delimiter">;; </span><span class="org-comment">TODO: Only update files that are generated by this file!</span> (dotcrafter-link-config-files))) (<span class="org-keyword">defun</span> <span class="org-function-name">dotcrafter--org-mode-hook</span> () (add-hook 'after-save-hook #'dotcrafter--after-save-handler nil t)) (<span class="org-keyword">defun</span> <span class="org-function-name">dotcrafter--after-save-handler</span> () (<span <span class="org-string">" dotcrafter"</span> <span class="org-builtin">:keymap</span> (list (cons (dotcrafter--key <span class="org-string">"t"</span>) #'dotcrafter-tangle-org-file) (cons
