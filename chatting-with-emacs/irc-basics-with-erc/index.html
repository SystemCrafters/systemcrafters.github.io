Chat Like the 90's in Emacs with ERC

<div><p><a href="https://www.youtube.com/watch?v=qWHTZIYTA4s">Watch the video</a> on YouTube! </p> <h2><a id="yet-another-thing-emacs-can-do" class="anchor" href="#yet-another-thing-emacs-can-do">¶</a>Yet another thing Emacs can do!</h2><p>If you haven't already heard, it's possible to do yet another thing in Emacs that you might already be doing on a daily basis using some other set of programs. We've already seen on this channel that you can use Emacs for your e-mail, so it probably won't come as a surprise that you can also use Emacs as a chat client! </p> <p>In this episode show you the basics of using the IRC chat protocol with Emacs' built-in client called ERC so that you can chat with other people from the comfort of your Emacs window! </p> <h2><a id="what-is-irc" class="anchor" href="#what-is-irc">¶</a>What is IRC?</h2><p>Internet Relay Chat, commonly known as IRC, is a protocol that enables real-time chat across a number of servers that are connected together to form something called an IRC network. You might have heard of Freenode, EFNet, DALnet, and many other popular IRC networks. </p> <p>If you're not familiar with IRC, you can think of it like a Slack or Discord server; you join a server on a particular IRC network and there are a bunch of channels that you can join to chat about a variety of topics. One difference though is that you must join the channels you want to participate in. You also don't see the history of a channel when you join it like you do in Slack or Discord; I'll show some ways to fix that in a future video. </p> <p>The nice thing about IRC is that it's an open protocol which means there are an incredible number of clients available for it on the Internet. What's even better is that can join a server right now with no account creation necessary using packages that are already included with Emacs! </p> <p>We're going to take a look at one of them today: ERC. </p> <h2><a id="what-is-erc" class="anchor" href="#what-is-erc">¶</a>What is ERC?</h2><p>ERC is a featureful and extensible IRC client written in Emacs Lisp. It's not the only IRC client that comes with Emacs (see rcirc) but it's probably the one that you'll want to use long-term if you stick with in-box clients. </p> <p><strong>Useful Features</strong> </p> <ul><li>Connect to multiple servers at once</li> <li>Can complete the names of other users while typing</li> <li>Highlighting for your name or other terms in messages</li> <li>Channel tracking to show channel activity in the mode line</li> <li>Show emojis in messages</li> <li>Desktop notifications when mentioned or messaged (enabled as a module)</li> <li>Easy to script it with new behavior!</li> </ul> <p>One nice thing about it is that you don't really even need to configure it to use it, you can run it right out of the box! </p> <p><a href="https://www.gnu.org/software/emacs/manual/html_mono/erc.html">ERC Manual</a> </p> <h2><a id="joining-a-server" class="anchor" href="#joining-a-server">¶</a>Joining a server</h2><p>Joining a server with ERC is very easy, you can just run <kbd>M-x erc-tls</kbd> to connect to a particular IRC server address and you'll be prompted for your "nickname" (the name you will use on the server) and your password. You can skip the password prompt since you may not have an account yet! </p> <p>You can also use <kbd>M-x erc</kbd> but I <strong>highly</strong> recommend connecting using <kbd>M-x erc-tls</kbd> if the server supports SSL/TLS to ensure that you make a secure connection to the server! This is very important for protecting the passwords you transmit to the server. </p> <p>I recommend joining the <a href="https://libera.chat/">Libera.Chat</a> server at <kbd>irc.libera.chat</kbd>. Official chat rooms for many free and open source projects you know are there (<kbd>#emacs</kbd>, <kbd>#guix</kbd>, <kbd>#sr.ht</kbd>) and you can also join the <kbd>#systemcrafters</kbd> channel! </p> <p>Let's do that now! </p> <h2><a id="joining-a-channel" class="anchor" href="#joining-a-channel">¶</a>Joining a channel</h2><p>Once you've connected to a server, you can join a channel by typing the following command: </p> <pre><code class="nil"> /join #systemcrafters </code></pre> <p>After you join the channel, a new buffer will appear in Emacs for that channel with the channel's name as the buffer name. You can switch to this buffer at any time to read the chat or send messages. Let's try typing a message to see if anyone responds! </p> <p>You'll probably notice that things look pretty minimal compared to more modern chat programs: no images, no profile icons, no emojis, etc. Some of these things can be added with extension modules, though! </p> <h2><a id="leaving-a-channel" class="anchor" href="#leaving-a-channel">¶</a>Leaving a channel</h2><p>If you ever want to leave a channel, you can use the <kbd>/part</kbd> command: </p> <pre><code class="nil"> /part </code></pre> <p>You can also leave a specific channel by passing the channel name to <kbd>/part</kbd>. </p> <h2><a id="listing-available-channels" class="anchor" href="#listing-available-channels">¶</a>Listing available channels</h2><p>You can get a list of all the channels on your IRC server by typing the <kbd>/list</kbd> command: </p> <pre><code class="nil"> /list </code></pre> <p>ERC will show a new buffer with a listing of all the available channels in the server. Since this is a normal Emacs buffer you can use your normal search key bindings to find channels with a particular string in the name or "topic." </p> <h2><a id="finding-more-details-about-a-user" class="anchor" href="#finding-more-details-about-a-user">¶</a>Finding more details about a user</h2><p>You can get a little bit more information about a particular user with the <kbd>/whois</kbd> command: </p> <pre><code class="nil"> /whois daviwil </code></pre> <p>This can return information like: </p> <ul><li>Their full name (if they provide it)</li> <li>The IRC node they are connected to</li> <li>The channels they are in</li> <li>Their IP address or "cloak"</li> </ul> <h2><a id="sending-someone-a-private-message" class="anchor" href="#sending-someone-a-private-message">¶</a>Sending someone a private message</h2><p>You can send someone a private message a couple of different ways. The most convenient way is to use the <kbd>/query</kbd> command: </p> <pre><code class="nil"> /query daviwil </code></pre> <p>This will open a new Emacs buffer for the private conversation between you and the recipient. You can type any message into this buffer and press RET to send the message. Replies from the user will also be written to the buffer. </p> <p>You can also use the <kbd>/msg</kbd> command to send a one-time message without opening a new buffer: </p> <pre><code class="nil"> /msg daviwil Hello there! </code></pre> <p>If the user responds, a query buffer will be opened! </p> <h2><a id="changing-your-nickname" class="anchor" href="#changing-your-nickname">¶</a>Changing your Nickname</h2><p>If you aren't satisfied with the name you chose when you joined the server, you can use the <kbd>/nick</kbd> command to change your name: </p> <pre><code class="nil"> /nick daviwil-x </code></pre> <p>Users across all the channels you're in will now see you as the new name! </p> <h2><a id="quitting-the-irc-server" class="anchor" href="#quitting-the-irc-server">¶</a>Quitting the IRC server</h2><p>If you'd like to quit the server, use the <kbd>/quit</kbd> command. You can also add a message which will be seen by everyone in the chat rooms you're currently in when you leave: </p> <pre><code class="nil"> /quit See you all later! </code></pre> <h2><a id="reconnecting-to-the-server" class="anchor" href="#reconnecting-to-the-server">¶</a>Reconnecting to the server</h2><p>If you want to reconnect to the IRC server, you can use the <kbd>/reconnect</kbd> command inside of a buffer for that server, like one of the chat windows: </p> <pre><code class="nil"> /reconnect </code></pre> <p><strong>NOTE:</strong> You might need to do this if you use a laptop and it wakes up from sleep! ERC doesn't do a good job of reconnecting reliably for me in this case so I have to reconnect manually pretty often. </p> <h2><a id="a-basic-erc-configuration" class="anchor" href="#a-basic-erc-configuration">¶</a>A basic ERC configuration</h2><p>Here's a basic ERC configuration you can use to get started: </p> <pre><code class="emacs-lisp"> (<span class="org-keyword">setq</span> erc-server <span class="org-string">"irc.libera.chat"</span> erc-nick <span class="org-string">"your-favorite-nick"</span> <span class="org-comment-delimiter">; </span><span class="org-comment">Change this!</span> erc-user-full-name <span class="org-string">"Emacs User"</span> <span class="org-comment-delimiter">; </span><span class="org-comment">And this!</span> erc-track-shorten-start 8 erc-autojoin-channels-alist '((<span class="org-string">"irc.libera.chat"</span> <span class="org-string">"#systemcrafters"</span> <span class="org-string">"#emacs"</span>)) erc-kill-buffer-on-part t erc-auto-query 'bury) </code></pre> <p>This will set your user details, default server, and channels to join upon connecting. It will also control how some buffers get created or closed when you leave channels or receive messages from other users! </p> <p>In future videos I'll show some modules you can enable to improve your experience even further! </p> <h2><a id="want-to-try-it-out--come-chat-with-us" class="anchor" href="#want-to-try-it-out--come-chat-with-us">¶</a>Want to try it out?
